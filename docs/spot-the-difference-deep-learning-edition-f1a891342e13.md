# 发现差异—深度学习版

> 原文：<https://medium.datadriveninvestor.com/spot-the-difference-deep-learning-edition-f1a891342e13?source=collection_archive---------1----------------------->

[![](img/d451fa7d522a366bea64dbd09c008fa0.png)](http://www.track.datadriveninvestor.com/1B9E)![](img/94f6534b3e9d33c6ae785a0b224507e5.png)

你玩过 ***点差*** ，对吧？(当然是你做的。我们*都*做到了😀).想一想当你试图找出不同之处时，你实际上做了什么——事实上，用上面的海绵宝宝杂耍再做一次。你可能会做一些像

1.  向左快速一瞥。
2.  向右快速一瞥
3.  *锅铲在左边。不知道右边有没有？*”
4.  "*没错，有。Sriracha 在右边。那是在左边吗？*”
5.  ”*不对！左边是番茄酱瓶子！而且它下面的盘子里什么都没有！*

等等。
总而言之，你基本上要做的就是交替看每张图像，识别每张图像中的物体，并试图在另一张图像中找到相同的物体。

事实证明，大多数深度学习系统所做的是完全不同的事情。他们基本上是这样做的

1.  找出第一幅图像中的所有东西(“*1 点钟方向的番茄酱瓶，3 点钟方向的空盘子，10 点钟方向的抹刀，…* ”)
2.  找出第二张图片中的所有东西(*1 点钟方向的 sriracha 瓶，3 点钟方向的星形盘，10 点钟方向的抹刀*)
3.  比较这两个列表，看看它们是否相同。

(*好吧，他们从技术上比较每张图片提取的特征向量，但那是不相关的。这，有点，有点像我上面说的*😑) (•)

为什么这很重要？好吧，当你在做一些*特定的*——例如[比较签名](https://papers.nips.cc/paper/769-signature-verification-using-a-siamese-time-delay-neural-network.pdf)——那么你可以训练你的 DL 系统来“*发现签名宇宙的差异*”，嗯，它就是工作。基本上，你训练你的 SignatureAnalysis 系统产生一个给定签名中所有内容的列表。这样，你可以给它两个签名，它就可以——很容易！—列出每个签名中的所有内容，然后进行比较。

OTOH，如果你给同样的 SignatureAnalysis 系统从上面看海绵宝宝的图像，它会完全丢失，对吗？这是因为，嗯，这几乎就是 DL 的工作方式——它知道如何在签名中找到独特的东西，*但是只有签名！你不会期望 SignatureAnalysis 与海绵宝宝一起工作，就像你不会期望 AlphaGo 能够在 DOTA 2 中获胜一样！*

当然，除非你能让深度学习系统变得更像人类。我的意思是，系统会学习如何进行比较，但它不会选择要比较的具体类型。所以，它会学习如何与 T2 比较 T3，而不是如何与 T4 比较 T5。元学习如果你愿意的话。但是说起来容易做起来难——事实证明，这真的很难做到，而且做起来非常脆弱。

这让我们最终关注*循环比较器*也称为 ARCs()。这些是一类深度学习模型，工作**迟缓**。一般来说，他们在*推理时间*(当你实际在玩“*发现差异*”)计算出他们的*表示空间*(基本上就是你正在看的东西是如何出现在 NeuralNetworkWorld 中的)。

![](img/6d75860e9fb12208aa751a5295f704ad.png)

或者，换句话说，如果你把上面的水果图片扔进去，系统就不走了”🤯🤯这些不是签名🤯🤯".相反，它基本上是经济特区。哼，看起来不像是签名，但不管怎样，我要找出其中的不同之处！

非常非常准确地说，[他们所做的是](https://arxiv.org/pdf/1703.00767.pdf)懒散地发展

> 仅在推理时以测试样本为条件的一种表示空间。在此之前，呈现给模型的所有样本都按原样存储在存储库中。当给定测试样本时，我们使用弧将该样本与我们库中的每个其他样本进行比较，以形成每个样本的相对表示(该表示是递归控制器的最终隐藏状态)。在相对于测试样本的相对表示空间中，给定相对表示空间的整个上下文，我们使用可以识别最相似样本对的训练分类器。这个相对表示空间是动态的，因为它相对于测试样本而变化。

如果你看着上面说“*哎！这基本上是一次性学习！“，你说得太对了！ARCs 允许你把一个系统训练在一组特征上(*这两辆* ***汽车*** *是一样的吗？*))并用它们来配合完全不同的东西(*这两只* ***猫*** *是一样的吗？*”)。*

请注意，和大多数类似的事情一样，天下没有免费的午餐。以这种方式做事要求当它发现差异时(DL 的*推理*部分)，它需要更多的计算能力。基本上，你可以在学习玩游戏的时候进行计算(*通过变得非常非常擅长签名*)或者在实际玩游戏的时候进行计算(*当你做水果、海绵宝宝或者其他什么的时候*)。
玩游戏时使用计算能力意味着，在 CPU/电池有限的手机上很难做到这一点，需要更多的时间，等等。

不过，这是一个非常漂亮的东西，不是吗？

()“ [*使用“暹罗”时间延迟神经网络*](https://papers.nips.cc/paper/769-signature-verification-using-a-siamese-time-delay-neural-network.pdf) 进行签名验证”Bromley 等人。
