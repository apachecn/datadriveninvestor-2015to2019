**1 —字符串:**

正如您可能已经知道的，输入验证是应用程序安全性的一个关键要素，您应该知道识别异常的最简单方法是您的数据/输入的“长度”。因此，在您的系统中，如果输入有一个预期的长度，或者如果我们可以预测它的最大长度(或多或少)，一个非常好的做法是为您的输入设置一个最大的边界。由于软件攻击需要大量的时间来利用系统，strlen 函数提供了一种非常经济有效的方式来保护您的系统，并且非常容易使用。

**2 — uint/ulong:**

好了，伙计们，作为一个开发者，你们应该知道 int 和 uint，long 和 ulong 之间的区别…等等。你可能听说过“u”代表 unsigned，但是如果你没有积极地使用它，也许你还没有意识到它的潜力。提醒你一下，因为你持有的价值没有符号，所以 uint 基本上有两个意思:

1-您可以存储更大的值

2-值不能为负数

就软件安全性而言，这可能在几个不同的方面有所帮助。首先，默认情况下，您将为您的输入设置一个“边界”,这将增加系统的稳定性。此外，这种实践还将帮助您提高数据类型意识。

边界限制在软件安全中很重要，strlen 和 uint/ulong 对创建安全的系统很有帮助。

**3 —了解你的类型—类型铸造:**

它对于业务逻辑、类型转换和类型转换都非常有用，是一个非常有用的风险缓解工具。在漏洞评估期间，我们已经看到了许多输入验证的坏例子，但是那些使用类型转换的不太可能受到影响，因为特定于语言的执行例程保护系统免受未知输入的影响。

因此，请始终将变量设置为相关的数据类型。在静态类型语言中，这是你默认的做法，但是懒惰的人脑倾向于将变量设置为字符串。了解你的类型！

**4 —默认值:**

在安全评估中，我们非常喜欢默认值。如果任何东西使用了默认设置，那么它更有可能被黑客攻击。你可能知道默认密码，但这不是我们正在谈论的。

如果您真的关心安全性，请尝试更改您使用的每个默认参数。考虑改变路径名(admin，wp_admin，files，logs…等)，数据库和表名(你可以添加一些前缀)，端口号，散列函数(使用盐)，关键片段和函数等。默认不好！

**5 —白名单:**

输入验证是一个痛苦的过程，并且严重依赖于您的业务逻辑。当然，没有一种通用的输入验证方法。但是，仍然有一些最佳实践可以应用；

首先，使用白名单而不是黑名单。找到一个“熟悉的”模式总是比试图过滤有害的输入更容易。

其次，考虑 strlen 技巧和类型转换。边界限制是白名单的很好例子。使用它们。

第三，对你的数据进行编码。在某些情况下，对输入进行编码可以防止注入攻击(默认情况下)。

第四，看看 ***JSON 模式验证*** 库。净化 JSON 对象可能非常有用。

第五，学习正则表达式。鉴于它们是如此强大和有用，难怪许多书都是关于它们的。即使你不能挖得这么深，至少知道一些基本知识。对于大多数输入，即使是 **\w** 过滤也能完美工作。

[\d] == >位数

[\w] ==> ASCII 可打印单词

[\s] == >空白

**6 —密码安全:**

安全密码存储很复杂！只有一个大的推荐。不要用明文存储密码！

**7 —搜索您的语言中的常见漏洞:**

在开始一个项目之前，寻找你的编程语言中“最常见的漏洞”。您可能不知道最近发布了一个特定于语言的严重漏洞。

这类漏洞的一个很好的例子就是 Java 和 C#中的反序列化漏洞。(反序列化漏洞也可能出现在 python、php、ruby 等其他语言中。)下面，您可能会找到“ysoserial”的链接，这是一个非常流行且严重的漏洞，会影响 Java 和 C#.NET。

[https://github.com/frohoff/ysoserial](https://github.com/frohoff/ysoserial)

[https://github.com/pwntester/ysoserial.net](https://github.com/pwntester/ysoserial.net)

另一个例子是 JavaScript 原型污染攻击，如果你使用 JavaScript 的话。

即使有很多关于软件安全的事情要说，如果你不确定如何开始编写安全的代码，你可以在开始一个新项目之前，使用这 7 个建议为自己创建一个清单。
