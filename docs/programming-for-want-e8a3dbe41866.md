# 为匮乏而编程

> 原文：<https://medium.datadriveninvestor.com/programming-for-want-e8a3dbe41866?source=collection_archive---------23----------------------->

[![](img/e712c537ea7e960cb7ed012f102d5631.png)](http://www.track.datadriveninvestor.com/1B9E)

你想要什么?

这是一个关键的问题，当从事物的消费者方面来看待人工智能时，这是一个至高无上的问题。首先出现在人们脑海中的人工智能，即我们在讨论即将到来的“机器人启示录”时开玩笑的人工智能，并不像我们用来瞄准广告或制造汽车的专门智能。这是一个更广泛、更“情绪化”的人工智能，能够预测它所纠缠的人类的想要和需要。这是一种人类形式的智能，完全有能力出于自身的原因说不。

但是我们建造东西不是为了听他们说他们不想要。

![](img/c356172dbb9acd2a339967e80ad4b840.png)

这种“情绪化”的人工智能是最难开发的一种，它可以知道你想要什么，而不是你要求什么。不是因为我们没有技术，也不是因为我们没有能够处理如此大量信息的计算机，而是因为我们根本没有时间。

时间是最重要的。

一个活生生的个人助理和一个具有相似功能的助理之间的最大区别是，一个活生生的人和你有相似的需求。我们不会有意识地想到的简单事情，比如理解零售商 B 的包装优于零售商 a 的包装。这意味着购买的商品更经常完好无损地到达，因此在价格上值得多花一美元。一个活的智能可以根据他们和你之间的相似性来预测你可能想要什么。这超出了“由肉制成”和“没有可呼吸的空气就会死亡”等基本假设。这涉及到理解共同的文化和经历、教育层次和社会经济差异。如果他们错了，那么他们可以被纠正，而这种纠正会波及到内部，并交叉应用到多项任务中。

与消费者人工智能的现状形成对比。像 Siri 和 Hey Google 这样的助手是非常任务驱动的，这是有充分理由的。随着时间的推移，他们可以了解你的偏好，但这是一个缓慢且不均衡的过程，而且这种学习还不能交叉应用。有趣的是，每一次互动都必须被视为教学时刻。作为消费者，你可能会说，“谷歌，我需要一个便宜的航班，本周五为我和孩子去波拉波拉，”并期待一个满意的结果。但是(正如我们现在可能都经历过的)你需要设置**非常**具体的参数。然后，你需要在事后仔细检查工作，这个过程很快就会达到这样一个点，那就是自己做会更快。半打这样的例子，你就放弃了，完全放弃了使用 AIsisstant。时间、精神和情感上的代价仍然太高。这种关系目前对于任何更高级的任务都是不成立的。

现在，如果这种情况确实(而且经常)发生在实时智能上，那么这个人可以并且将会观察你的交易，这样他们就有了一个既定的工作框架。你不必直接教他们，允许或鼓励观察往往就够了。

注意，我说的是*从*开始工作。这是关键。以人工智能目前的状态，一旦你在一项任务中训练它们，它们就能精确地复制你想要的次数。*但是*如果这项任务的任何条件发生变化，他们就无法适应。即使我已经在 50 次在线预订的过程中训练了我的助手，任何新的变量都意味着训练必须从头开始。他们目前还没有能力进行横向思维，而这种思维需要更多的帮助，而不仅仅是清单的执行者。

这就是消费级人工智能目前状况的问题所在；一个活着的智能能够理解想要什么。你想要一个栖身之所，你想要一个芝士汉堡而不是甘蓝沙拉。没有这种联系，你将很难开发出一个能给你想要的而不是你要求的人工智能。它将适用于重复性的服务任务，但永远不会达到我们想象的灵活的、人类形式的智能。

从全局来看，这可能不是最坏的结果。将机器引入我们生活的目标一直是效率。它从来没有取代我们，虽然在许多任务中他们这样做了。最终目标是解放我们。把我们从接触有毒化学物质的劳动中解放出来，把我们从因不含咖啡因的错误而导致生命或肢体损失的工作中解放出来。也许最好的目标是专注于开发更简单的人工智能，使我们的生活更容易，同时仍然把所有更大的决定留给我们。
