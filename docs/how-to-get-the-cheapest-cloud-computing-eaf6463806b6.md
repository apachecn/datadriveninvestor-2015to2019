# 寻找能够满足您需求的最便宜的云计算提供商

有各种各样的小型云提供商试图通过降低价格来竞争。如果你为小企业工作，喜欢简单的体验，也许其中一个适合你。

然而,“三大”云提供商——亚马逊网络服务(AWS)、微软 Azure 和谷歌云— [主宰市场是有原因的。他们提供广泛的产品线，并不断创新。他们的停机频率很低，而且他们的规模需要简单的入职流程和大量的文档。](https://www.parkmycloud.com/blog/aws-vs-azure-vs-google-cloud-market-share/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=aws-vs-azure-vs-google-cloud-market-share)

无论您决定选择哪家提供商，都要确保您能够获得所需的所有服务—是否有计算产品、存储、数据库？如果您想使用容器或选择无服务器，这些产品如何适合？客户支持有多好？你的公司是否直接与供应商竞争——例如，与亚马逊的零售部门竞争？(你可能不在乎，但有些公司[肯定做](https://www.parkmycloud.com/blog/aws-alternatives/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=aws-alternatives)。)

虽然在主要选项中没有一个“最便宜”的云提供商，但您仍然应该进行比较，以确保您获得最常用服务的最佳云价格。有关三大提供商定价的更多信息，请参见以下云计算成本比较:

*   [AWS vs 谷歌云定价](https://www.parkmycloud.com/blog/aws-vs-google-cloud-pricing/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=aws-vs-google-cloud-pricing)
*   [AWS 与 Azure 定价](https://www.parkmycloud.com/blog/azure-cloud-pricing/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=azure-cloud-pricing)
*   [云提供商之间的差异到底有多重要？](https://www.parkmycloud.com/blog/cloud-providers/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=cloud-providers)

关于供应商锁定的想法:如果你已经从一个云服务提供商那里购买云服务，你可能会担心你被该提供商“锁定”。我们在实践中看到的情况略有不同:随着按需灵活性和比以往更多的机会来实践云计算，[公司不应该真正担心](https://www.parkmycloud.com/blog/vendor-lock-in/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=vendor-lock-in)在公共云方面的供应商锁定。

# 如何从您当前的提供商那里获得最便宜的云计算

当然，无论您是否担心供应商锁定，您都应该确保您获得最有效的云计算成本。这意味着优化您最常用产品的选项。

这里有一个你应该做的事情的简要概述，以确保你从你当前的提供商那里得到最便宜的云计算。

# 为生产环境使用保留实例

所有主要的云提供商都为保留的实例提供定价选项，也就是说，如果您承诺随着时间的推移使用容量，您可以支付折扣价。保留实例可以省钱——只要你正确使用它们。对于保留的实例，重点关注具有 24×7 需求的工作负载(即生产工作负载)非常重要。最长的承诺，你会得到最好的价格。当然，每个云提供商对此选项的结构都不同。以下是我们的指南:

*   [AWS 保留实例](https://www.parkmycloud.com/blog/are-aws-reserved-instances-better-than-on-demand/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=are-aws-reserved-instances-better-than-on-demand)
*   [Azure 保留实例](https://www.parkmycloud.com/blog/azure-reserved-instances/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=azure-reserved-instances)
*   [谷歌承诺使用折扣](https://www.parkmycloud.com/blog/google-cloud-committed-use/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=google-cloud-committed-use)

# 只为你实际需要的东西付费

有一些常见的方式会让用户无意中浪费金钱，放弃最便宜的公共云账单，例如使用比他们需要的更大的实例，以及全天候运行开发/测试实例，而不是仅在需要时运行。为了满足您的需求，请确保您的所有实例都“大小合适”,达到与工作负载最匹配的大小。您还应该使用开/关时间表，以便在晚上和周末关闭用于开发、测试和试运行的非生产资源。

# 利用其他折扣价格选项

主要的云提供商提供了许多其他折扣价格和购买选项，以帮助您获得最便宜的云服务。

*   [AWS Spot 实例](https://www.parkmycloud.com/blog/aws-spot-instance/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=aws-spot-instance) —获得最便宜 EC2 实例的最佳方式。该选项为多余的基础架构提供了很大的折扣，这些基础架构可以随时回收用于其他工作负载。
*   [Azure 低优先级虚拟机](https://www.parkmycloud.com/blog/azure-low-priority-vms/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=azure-low-priority-vms)——类似于 AWS 的 spot 实例，尽管 Azure 的产品有固定的折扣和一些其他的操作差异。
*   Google Cloud 可抢占虚拟机 — Google Cloud 的可抢占虚拟机是另一个类似的选项。
*   [谷歌云持续使用折扣](https://www.parkmycloud.com/blog/google-sustained-use-discounts/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=google-sustained-use-discounts) —这是谷歌用户的内置定价机制，每台服务器的运行次数越多，价格就越低。
*   [Azure 开发/测试定价](https://www.parkmycloud.com/blog/azure-dev-test-pricing/?utm_medium=referral&utm_source=medium&utm_campaign=medium blogs&utm_content=azure-dev-test-pricing) —专门针对用于开发和测试的工作负载的折扣选项。
