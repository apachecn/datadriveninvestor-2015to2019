**机器学习系统的类型:**

机器学习领域相当广阔，并且往往有不同的“学习”范例或系统。根据我们试图解决的问题类型，每个系统都有其使用案例。在这个故事中，我简要地讨论了其中的一些系统。无论如何，这不是一个详尽的列表，但它涉及了我在学习或实现机器学习时看到的事情。

**监督学习**

这是我遇到的最常见的一种系统。在监督学习中，我们有训练数据集，其中包括问题的“期望”解决方案。这些解决方案通常是由人类专家创造的。我们可以进行两种类型的学习:分类和回归。

**无监督学习**

在这种学习中，我们只被提供了训练数据。没有“想要的”标签。机器学习算法试图在没有任何监督的情况下找到数据中的潜在模式。聚类和降维是属于这种范式的一些常见技术。

**强化学习**

这是新的炒作！以前，这个领域被认为是非常理论化的，但近年来，研究界在使这些系统更加实用方面取得了长足的进步。在强化学习中，代理人或模型通过“强化”的过程学习最大化回报。这类似于我们学习的方式。这需要大量的反复试验。一些著名的例子有 AlphaGo/Zero、OpenAI5、DQN(Atari)。

**基于实例与基于模型的对比**

这不是阿瑟的范式/系统，而是一种技术。在基于实例的学习中，我们不专注于寻找模型，而是基于相似性进行学习。基于实例的学习器本质上只是存储数据，没有参数调整。他们也被称为懒惰的学习者。

另一方面，基于模型的学习包括通过创建模型进行归纳的过程。我们试图模拟环境/问题。随着这种“建模”，我们试图找到最佳参数，使我们能够将这些模型推广到各种问题。

**批量学习 vs 在线学习**

和以前一样，这些不是真正的系统，更像是技术。批量学习是指使用整个数据集来训练模型。这通常很耗时，并且是“离线”完成的。

另一方面，在线学习本质上更具有递增性。这是在单个训练样本/小批量样本上进行的。在线学习是一个“实时”的过程。
